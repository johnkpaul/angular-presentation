Templates

Angular has a very expressive template system, which looks just like HTML and adds custom elements.

Templates - Interpolation

Double curlies include properties from the component in the output

You can use back ticks ` ` in your template to use single or double quotes inside.

Templates - Interpolation

Simple expressions are also allowed, you can run a component method (like fullName() below).

Templates / Interpolation / Properties

When giving a value to the element's attribute, you can use the syntax {{ componentPropertyName }}

Templates - Property Binding

An other option is to use property binding [attribute] = property

By doing property binding you do not need "{{ curlies }}" to specify your property.

Templates / Data binding extras

Angular supports more advanced bindings than just attribute name

Event binding: (event)

The bindings directives you've met so far flow data in one direction: from a component to an element

Reference binding ( #var )

Sometimes we need to access a component from the template. We can mark it with #name, and it becomes available as name everywhere in the template

Conditional Display (*ngIf)

This conditional expression will add or remove an element from the DOM if it evaluates as a truthy.

Add ! before the onDisplay() function, to get a truthy result.

Conditional Display (*ngFor)

Loops over items in a collection and adds a template to the DOM for each item in the collection.

The *ngFor directive will display one puppy name for every puppy in the list.

Note the * that identifies a DOM structure-altering directive and the camelCase!
